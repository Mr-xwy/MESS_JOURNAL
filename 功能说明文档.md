> 适用对象：第一次接触本项目的开发者 / 团队成员

> 配套文档：本文档讲「功能与模块」；[接口文档.md](./接口文档.md) 讲「每个接口的入参/出参细节」。

## 1. 项目概述

M.E.S.S Journal 是一个**学术期刊风格的博客系统**（参考 S.H.I.T Journal）。后端基于 Spring Boot 3，提供用户、文章、热点、AI 助手四大模块；前端是一套纯原生 HTML/CSS/JS 页面，由后端同源托管（无需独立前端工程，直接访问 `http://localhost:8080/` 即可）。

设计定位：
- **用户体系**：注册 / 登录 / 注销，JWT 鉴权 + 拦截器，密码 MD5 加盐，三级权限（ADMIN / MODERATOR / USER）。
- **文章体系**：分页列表、分类、关键词搜索、草稿箱、发布、编辑、删除、下线/恢复、点赞。
- **热点体系**：已发布文章按点赞量倒序的热度榜。
- **AI 助手**：右下角浮窗，由独立的 Python + LangChain 智能体驱动（记忆 + 自动工具调用：站内检索、天气查询），Spring Boot 仅做代理转发。

## 2. 技术栈

| 维度 | 选型 |
| --- | --- |
| 后端框架 | Spring Boot 3.2 + Spring MVC |
| 持久层 | Spring Data JPA (Hibernate) |
| 数据库 | MySQL（表结构由 JPA `ddl-auto: update` 自动生成，无需手动建表） |
| 安全 | JWT（jjwt 0.12）+ 自定义 `HandlerInterceptor` 拦截器 |
| 密码 | MD5(`密码` + `随机盐`)，盐随用户生成 |
| 前端 | 原生 HTML / CSS / JS，无构建工具 |
| AI | 独立 Python + LangChain 微服务（ai-assistant，端口 8000），由 Spring Boot 代理转发；具备近 10 轮记忆与工具调用 |
| 统一返回 | `Result<T>`（code/message/data），分页用 `PageResult<T>` |

## 3. 快速开始

### 3.1 环境要求
- JDK 17+
- 本机 MySQL（默认连接 `localhost:3306/blog`，账号 `root` / 密码见 `application.yml`）

### 3.2 启动
1. 用 IDEA 打开 `blog-springboot` 工程，等待 Maven 依赖下载完成。
2. 确认 `src/main/resources/application.yml` 中 MySQL 账号密码与你本机一致。
3. 运行 `BlogApplication` 主类。
4. 浏览器访问 `http://localhost:8080/`（首页 `index.html`）。

> 数据库无需预建：连接串带 `createDatabaseIfNotExist=true`，且 JPA 会自动建表。首次启动会由 `DataInitializer` 写入种子账号与示例文章。

### 3.3 预置账号（可直接登录体验权限差异）

| 用户名 | 密码 | 角色 | 能力 |
| --- | --- | --- | --- |
| `admin` | `admin123` | ADMIN | 最高权限：可改/删任何人文章、可下线/恢复 |
| `moderator` | `mod123` | MODERATOR | 可下线/恢复他人文章（内容审核），不能删 |
| `alice` | `user123` | USER | 普通用户：仅管理自己的文章、可点赞 |
| `bob` | `user123` | USER | 普通用户 |

## 4. 模块与功能清单

> 每个功能点后面标注对应的接口路径，详细入参见《接口文档.md》。

### 模块一：用户与鉴权模块（前缀 `/api/auth`、`/api/user`）

| 功能 | 说明 | 对应接口 |
| --- | --- | --- |
| 用户注册 | 用户名 + 密码必填，昵称/邮箱选填；密码 MD5 加盐存储；默认角色 USER | `POST /api/auth/register` |
| 用户登录 | 校验密码，成功后返回 JWT（`token`）与脱敏用户信息 | `POST /api/auth/login` |
| 用户注销 | 将当前 token 的 `jti` 加入黑名单，使其立即失效 | `POST /api/auth/logout` |
| 获取我的信息 | 返回当前登录用户的脱敏资料 | `GET /api/user/info` |
| 修改我的资料 | 可改昵称 / 邮箱 / 头像；改密码需校验原密码 | `PUT /api/user/info` |

**鉴权要点**
- 前端 `js/api.js` 会自动在请求头附加 `Authorization: Bearer <token>`。
- 公开接口（无需登录）：登录、注册、注销、`GET /api/articles*`（只读浏览）、`GET /api/hot*`（只读浏览）。
- 其余 `/api/**` 必须携带合法 token，否则返回 `401 未登录或 token 缺失`。
- 文章列表/详情为「半公开」：无 token 可匿名浏览；带 token 时后端注入用户信息，用于「只看我的」(`mine=true`)、是否已点赞等个性化逻辑。

### 模块二：文章模块（前缀 `/api/articles`）

| 功能 | 说明 | 对应接口 |
| --- | --- | --- |
| 文章列表 | 分页 + 分类筛选 + 关键词搜索；`mine=true` 只看自己的（含草稿）；公开时仅返回已发布 | `GET /api/articles` |
| 文章详情 | 返回文章正文 + 当前用户是否已点赞；草稿仅作者可见，已下线仅作者/审核可见 | `GET /api/articles/{id}` |
| 发布 / 存草稿 | 登录后发布文章；`draft=true` 存为草稿，否则直接发布 | `POST /api/articles` |
| 编辑文章 | 作者本人或 ADMIN 可改；草稿改发布时自动补全发布时间 | `PUT /api/articles/{id}` |
| 删除文章 | 作者本人或 ADMIN 可删（审核员 MODERATOR 只能下线，不能删） | `DELETE /api/articles/{id}` |
| 下线 / 恢复 | 仅 MODERATOR 与 ADMIN；下线后文章对普通用户不可见 | `PATCH /api/articles/{id}/status` |
| 点赞 / 取消 | 登录后 toggle 点赞；按用户去重；仅已发布文章可点赞 | `POST /api/articles/{id}/like` |

**文章状态机**（`ArticleStatus`）
- `DRAFT` 草稿：仅作者本人可见（在「我的草稿箱」）。
- `PUBLISHED` 已发布：所有人可见、可点赞、可上热点榜。
- `OFFLINE` 已下线：被审核员/管理员处理，仅作者与审核人员可见。

**权限矩阵（文章写操作）**

| 操作 | USER | MODERATOR | ADMIN |
| --- | --- | --- | --- |
| 发/改/删自己的文章 | ✅ | ✅ | ✅ |
| 改/删他人文章 | ❌ | ❌ | ✅ |
| 下线/恢复他人文章 | ❌ | ✅ | ✅ |

### 模块三：热点模块（前缀 `/api/hot`）

| 功能 | 说明 | 对应接口 |
| --- | --- | --- |
| 热点榜 | 已发布文章按点赞量倒序（点赞相同按发布时间倒序）；`limit` 控制条数，默认 10 | `GET /api/hot` |

公开只读，无需登录。

### 模块四：AI 助手模块（前缀 `/api/ai`）

| 功能 | 说明 | 对应接口 |
| --- | --- | --- |
| AI 问答 | 前端提交问题，Spring Boot 转发给独立的 Python + LangChain 微服务，返回回答文本 | `POST /api/ai/chat` |

- 需登录（避免匿名消耗外部额度）。
- **架构**：前端同源调用 `POST /api/ai/chat`（带 JWT）→ Spring Boot `AiController` 把登录用户名作为 `session_id` 转发给 Python 服务 `http://localhost:8000/api/ai/chat` → Python 的 LangChain 智能体调用 OpenRouter（密钥/模型仍来自 `application.yml` 的 `openrouter` 段）。前端零改动。
- **记忆能力**：每个登录用户保留最近约 10 轮问答（`ConversationBufferWindowMemory(k=20)`，每轮 = 人类 + AI 两条消息）；未登录用户归到 `default` 会话。
- **自动工具调用**：智能体会自行决定调用
  - `search_website`：检索本站前端源码（HTML/JS/MD），回答“在哪里修改密码”“怎么发布文章”等网站导航问题；
  - `get_weather`：调用 Open-Meteo 免费接口（无需 Key）查询任意城市实时天气。
- 外部大模型密钥在 `application.yml` 的 `openrouter.api-key`（**不写在前端、不写死在代码里**）；Python 服务地址在 `ai.python-url`（默认 `http://localhost:8000/api/ai/chat`）。
- 免费模型响应偏慢，超时已放宽（连接 10s、读取 120s）。

## 6. 统一返回结构

所有接口返回 JSON，结构一致：

```json
{
  "code": 0,        // 0 = 成功；非 0 = 失败（含 401/403/业务码）
  "message": "ok",  // 成功为 "ok"，失败为错误描述
  "data": { }       // 业务数据；分页接口为 PageResult 结构
}
```

分页接口 `data` 形态（`PageResult`）：

```json
{
  "list": [ /* 当前页数据数组 */ ],
  "total": 100,      // 总条数
  "page": 0,         // 当前页（从 0 开始）
  "size": 8,         // 每页条数
  "totalPages": 13   // 总页数
}
```

## 8. 给新人的几条提示

1. **改了前端 JS/CSS**：硬刷新（`Ctrl+Shift+R`）即可，已在 `application.yml` 关闭静态资源缓存。
2. **改了 Java**：必须重启后端。
3. **想让 AI 浮窗真正能聊**：先在 `application.yml` 的 `openrouter.api-key` 填入 Key（申请地址见文件注释），再启动 Python 服务（启动步骤见 `E:\Ai生成\ai-assistant\README.md`），最后重启 Spring Boot。
4. **权限想全体验**：直接用预置的 `admin` / `moderator` / `alice` 三个账号对比操作。
5. **接口联调**：所有接口前缀 `http://localhost:8080/api/...`，除登录注册注销和只读文章/热点外，都需要在 Header 带 `Authorization: Bearer <token>`。
