# 杂研 · M.E.S.S Journal 项目优化方案（资深开发审计）

> 审计对象：`blog-springboot`（SpringBoot 3.2 + MySQL + JPA + 原生前端，报纸学术期刊风）
> 审计方式：实际通读 64 个 Java 文件、19 个前端 JS、application.yml、拦截器与公共脚本
> 审计目标：找出各模块可优化的点，给出分优先级、可落地的优化方案，并同步提升团队工程能力

---

## 一、总体结论

**架构健康度：B+（结构清晰、分层合理、关键防护已就位）**
- 亮点的工程实践：JWT 半公开白名单、`GlobalExceptionHandler` 统一异常、`escapeHtml` 已定义且实现完整、Service 层事务边界基本正确、`ai-assistant` 双模式容错。
- 但存在 **P0 级安全/正确性/可维护性隐患**，且**零自动化测试**，距离"可被团队接手、可上生产"还有一段路。

**最该先解决的 5 件事（P0）**
1. 密码用**无盐 MD5** 存储（MD5Util 自己都注明不安全）
2. **零测试**（无 `src/test`），回归与协作无保障
3. CORS `allowedOriginPatterns("*")` + `allowCredentials(true)`（任意源可带凭证）
4. AI 会话历史用 `ConcurrentHashMap` 但**只按会话裁剪、不按用户数上限/TTL** → 长驻服务内存泄漏
5. AI 调用用**同步 `RestTemplate` + 120s 读超时**阻塞 Tomcat 线程 → 并发下线程池打满

## 二、全局 / 跨模块问题（最高优先级）

| 优先级 | 问题 | 现状证据 | 优化方案 | 团队提升点 |
|---|---|---|---|---|
| **P0** | 密码无盐 MD5 | `Md5Util.md5()` 无盐；注释已自承不安全 | 换 **BCrypt**（`PasswordEncoder` + 随机盐），登录/注册/改密全链路替换 | 安全基线：密码存储规范、彩虹表风险 |
| **P0** | 零自动化测试 | 无 `src/test` 目录 | 引入 **JUnit5 + MockMvc**，先覆盖 AI/Shop/Article 核心链路与鉴权拦截器 | 测试文化：TDD 入门、Mock 思维 |
| **P0** | CORS 任意源+凭证 | `WebConfig` `allowedOriginPatterns("*")`+`allowCredentials(true)` | 改为**白名单**（localhost:8080/你的域名），生产关掉 credentials 或仅放行受信源 | 安全：CORS/CSRF 认知 |
| **P0** | AI 会话内存泄漏 | `AiService.sessionHistories` 仅 `MAX_HISTORY_ROUNDS` 裁剪，用户数无上限无 TTL | 用 **Caffeine**（带 TTL/最大条数）或 **Redis** 存会话；或定时清理空闲会话 | 内存管理、缓存框架 |
| **P0** | AI 同步阻塞 | `RestTemplate` 读超时 120s，占满请求线程 | 改 **WebClient（响应式）** 或异步 + 超时；优先**流式输出**（SSE）提升体感 | 响应式编程、背压、超时治理 |
| **P1** | 图片 base64 入库 | `User.avatar`/`Relic.image`/`Product.image` 为 MEDIUMTEXT，列表接口整条返回 | 改**文件存储**（你已接 OSS/可加 MinIO）+ DB 仅存 URL；列表 DTO 不返回 base64 | 存储选型、DTO 投影 |
| **P1** | 无 DB 版本管理 | `ddl-auto: update` | 引入 **Flyway/Liquibase**，生产用 `ddl-auto: validate` | 数据库迁移、可重复部署 |
| **P1** | XSS 靠自觉 | `escapeHtml` 已定义，但靠人记；`crossword.js`/`puzzle.js` `escapeHtml` 计数 < `innerHTML` | 建立**强制策略**：用户内容一律 `escapeHtml`；富文本用 **DOMPurify(前端)/JSoup(后端)** 净化，绝不裸 `innerHTML`；加代码评审清单 | 前端安全、输入净化 |
| **P1** | DTO 校验不全 | 仅 Article/Letter/Login/Order 有 `@NotBlank`；分页 `size` 无上限 | **所有 DTO 加 `@Valid` + 约束**（`@Size`/`@DecimalMin` 价格非负/标题长度）；Controller 入参加 `@Valid`；分页 `size` 封顶（如 ≤50） | Bean Validation、防御式编程 |
| **P1** | 热点榜无缓存 + 模糊搜索慢 | `ArticleService.hot()` 每次查库；`list()` 用 `LIKE '%content%'` 全表扫 | 热点榜 `@Cacheable`（短 TTL）；搜索改 **MySQL FULLTEXT** 索引 | 缓存、全文检索 |
| **P1** | 内部接口暴露 | `JwtInterceptor` 放行 `/api/ai/query`（注释"仅本机 Python"），但实际公网可达 | 若 Python 代理不再使用则**下线该端点**；否则限制来源/不公开 | 攻击面收敛 |
| **P2** | 无日志/监控/CI | 无结构化日志、无 Dockerfile、无流水线 | 加 **SLF4J MDC**（链路 ID）、健康探针；补 `Dockerfile` + GitHub Actions 基础 CI | 可观测性、DevOps |
| **P2** | 前端无构建/无 Lint | 原生 JS + `?v=` 手动版本；无 ESLint | 引入 **ESLint + Prettier**，必要时 Vite 打包；统一 `api.js`/工具函数 | 前端工程化 |

## 三、分模块优化点

### 1. 博客文章模块（Article）
- **亮点**：`Specification` 动态查询、`detail()` 做了下线/草稿可见性控制、`toggleLike` 去重正确。
- **可优化**：
  - 列表接口返回整个 `Article` 实体（含 `content`，若文章内嵌 base64 图会严重膨胀响应）→ 用**投影/摘要 DTO**（id/title/summary/author/category/likeCount/status）只取列表所需字段。
  - 关键词搜索 `LIKE '%keyword%'` 命中 `content` 大字段 → 慢；改为 FULLTEXT（见全局 P1）。
  - 发布无字数/频率限制 → 可加`@Size` 与简单频率防护。
  - `update` 仅当 `content!=null` 才更新，但 `title/summary/category` 为 null 时保持旧值，逻辑可接受，建议补单测固化行为。

### 2. 热点榜（Hot）
- **可优化**：`@Cacheable` 缓存（见全局 P1）；列表与文章列表复用同一实体投影，避免重复 SQL。

### 3. 线上博物馆（Relic）
- **可优化**：`Relic.image` 为 MEDIUMTEXT base64 → 改文件存储（全局 P1）；列表页同样用投影 DTO，避免 base64 进列表响应。文物详情的 SVG/图片展示建议走 CDN/对象存储 URL。

### 4. 字谜 & 每日谜题（Puzzle / Crossword）
- **可优化**：
  - `crossword.js`（`innerHTML=7, escapeHtml=3`）、`puzzle.js`（`innerHTML=6, escapeHtml=2`）**转义覆盖率偏低**，虽数据为游戏词/服务器生成，仍需逐处确认无用户可控内容直接注入。
  - 每日谜题词方可**预生成并缓存**（每日固定一份），避免每次重新计算。
  - 计时逻辑（`puzzle.js`）已做"点击后开始"，可补单测固化状态机（`pending→started→submitted`）。

### 5. 版画工坊（Engraving）
- **现状**：本审计未改动该模块，前端 `engraving.js` 无 `innerHTML`（风险低）。
- **可优化**：若涉及用户上传图，统一走文件存储；导出功能建议后端生成、前端仅触发下载，减轻浏览器负担。

### 6. 读者来信（Letter）
- **可优化**：`LetterRequest` 已加 `@NotBlank`，好；列表/详情同样建议用投影 DTO；来信内容渲染务必 `escapeHtml`（前端来信页需复查，来信类内容常被忽略转义）。

### 7. AI 策展人 & AI 助手（AiService / AiController）
- **亮点**：双模式容错（直连 OpenRouter + 回退 Python）、按用户隔离历史、异常兜底。
- **可优化（最高价值）**：
  - **内存泄漏 + 阻塞**（全局 P0）必须改：Caffeine/Redis + WebClient 异步/流式。
  - **流式输出**：当前一次性返回，长文体感差；改 **SSE**（`text/event-stream`）逐字推送，前端打字机效果，体验质变。
  - **成本与限速**：建议加单用户每分钟调用上限、最大上下文长度裁剪、模型可配置。
  - `/api/ai/query` 公开暴露（全局 P1）。
  - 接你后续的 **RAG/向量检索**：站内文章/文物做嵌入，让"AI 一句话导读/策展"基于自有语料，而非纯 prompt（呼应你的 LangChain→RAG 学习路线）。

### 8. 古董铺（Shop）—— 最该补"正确性"
- **亮点**：库存锁定（`createOrder` 立即 `SOLD`）、越权拦截（卖家/管理员）、待付订单拦截下架。
- **可优化（正确性）**：
  - **订单号有碰撞风险**：`genOrderNo` 用 `currentTimeMillis()%1e6 + 随机`，并发可能重复 → 改 **雪花 ID / 数据库序列 / UUID**，或 `orderNo` 加唯一索引。
  - **库存永久锁定**：付款按钮置灰期间，商品下单即 `SOLD`、无释放机制 → 加**待付订单超时释放**（定时任务/Cron 把超时 PENDING 回滚为 AVAILABLE），否则弃单会永久占库存。
  - **接真实支付**：目前 `payment.html` 按钮置灰，后续用你熟悉的 FastAPI/Java 支付网关替换，并把"标记 SOLD"挪到 `PAID` 之后。
  - 列表整条返回 `Product`（含 base64 图）→ 投影 DTO（全局 P1）。

### 9. 用户与鉴权（Auth / User / Jwt）
- **可优化**：
  - **密码 BCrypt**（全局 P0）。
  - 登录无**限流/锁账户** → 加失败次数限制，防爆破。
  - `TokenBlacklist` 已支持注销，建议明确 token 过期策略（access/exp）。
  - 头像 base64 → 文件存储（全局 P1）。

### 10. 前端体验（共享 style.css / common.js / 各页 JS）
- **亮点**：已做入口页、暗墨对比、自定义光标（已按 reduced-motion/触控禁用）、滚动揭示、卡片微交互——整体高级感已成型。
- **可优化**：
  - **XSS 强制化**（全局 P1）：把 `escapeHtml` 作为唯一出口；富文本场景引入净化库。
  - **复用收敛**：19 个页 JS 中列表渲染/空态/错误态大量重复，可抽公共 `renderList()`/`EmptyState` 组件（轻量，不必上框架）。
  - **性能**：自定义光标是 `requestAnimationFrame` 线性插值，开销小；但大量 `box-shadow`/backdrop-filter 在低端机可能掉帧，建议对 `prefers-reduced-motion` 用户进一步降级。
  - **缓存策略**：`?v=` 手动版本易漏刷（此前多次出现旧 CSS 不生效），建议 Vite 构建自动哈希。

## 四、团队技术能力提升路线（结合你的 Java 全栈 + Python AI 路线）

1. **建立代码评审（Code Review）基线**
   - 必查清单：入参是否 `@Valid`、用户内容是否 `escapeHtml`/净化、DB 是否返回 base64、事务边界、越权判断。
   - 用 GitHub PR + 至少 1 人评审通过才合入。

2. **测试文化（从 0 到 1）**
   - 第一周：给 `AiService`（Mock RestTemplate）、`ShopService`（库存/订单号/越权）、`JwtInterceptor` 写单测。
   - 第二周：Controller 层 `MockMvc` 集成测试覆盖主要接口。

3. **工程化工具**
   - Java：Spotless + Checkstyle + SonarLint（本地）、GitHub Actions 跑 build+test。
   - 前端：ESLint + Prettier。

4. **安全基线培训**
   - 密码存储（BCrypt）、XSS（转义 vs 净化）、CORS/CSRF、JWT 过期与注销、最小权限。

5. **与学习路线结合（让项目成为练手场）**
   - **RAG 站内语义搜索**：把文章/文物做向量化，用你学的 LangChain+RAG 替代 `LIKE` 模糊搜索（直接解决全局 P1 搜索慢）。
   - **FastAPI 包装 AI 能力**：把 AiService 的 OpenRouter 调用抽到你学的 FastAPI 微服务，Java 后端只调 HTTP（呼应"FastAPI 是最后一层壳"的路线铁律）。
   - **Agent 化策展**：用 Agent/Function Calling 让 AI 自动从站内语料策展专题。

## 六、可立即执行的"第一周清单"（最小改动、最大收益）

1. `Md5Util` → `BCryptPasswordEncoder`（约 30 行改动，覆盖登录/注册/改密）。
2. `WebConfig` CORS 改为白名单（约 5 行）。
3. `AiService` 的 `sessionHistories` 换成 Caffeine（`Caffeine.newBuilder().expireAfterAccess(30m).maximumSize(5000)`）。
4. 给 `ShopService.genOrderNo` 加唯一约束（UUID/雪花）。
5. 新建 `src/test`，写 `JwtInterceptorTest` + `ShopServiceTest` 两个用例，跑通 CI 雏形。
6. 在 `common.js` 顶部加注释规范："所有用户内容渲染必须经 `escapeHtml` 或净化库"，并复查 `crossword.js`/`puzzle.js`/`letters.js` 的注入点。
