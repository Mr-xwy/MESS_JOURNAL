# 功能模块介绍（团队答辩用）

> 本文档面向**团队内部与答辩评审**，帮助快速理解项目「有哪些模块、各自做什么、技术亮点在哪、怎么演示、如何分工」。
> 配套完整 API 与运行说明见同目录 **README.md**。

---

## 0. 项目一句话介绍

**杂研 · M.E.S.S Journal** 是一个基于 **Spring Boot + MySQL + JWT** 的全栈「报纸学术期刊风」内容社区，在基础博客之上集成了**线上博物馆、读者来信、每日谜题、版画工坊、古董铺闲置商城、AI 策展人** 等模块，覆盖「内容生产 → 审核 → 互动 → 交易 → AI 增强」的完整闭环，适合作为课程设计 / 团队分工演练的综合性案例。

---

## 1. 整体架构（给非后端同学）

```
┌─────────────────────────┐         HTTP / JSON          ┌──────────────────────────┐
│   浏览器（静态前端）        │  ───── fetch('/api/...') ──▶  │   Spring Boot 后端          │
│  HTML/CSS/JS 同源托管 :8080 │  ◀──── Result<T> JSON ─────  │  Controller→Service→JPA    │
│  api.js / common.js        │                             │  ↕ MySQL (blog 库)          │
│  JWT 存 localStorage       │                             │  AiService → OpenRouter/Python│
└─────────────────────────┘                             └──────────────────────────┘
```

- **同源托管**：前端文件放在 `src/main/resources/static/`，由 Spring Boot 直接伺服，无需 Nginx / 独立前端工程。
- **无状态鉴权**：登录后前端持有 JWT，每个请求头带 `Authorization: Bearer <token>`，后端 `JwtInterceptor` 统一校验。
- **前后端契约**：统一返回 `Result{code,message,data}`，`code===0` 即成功。

## 2. 角色与权限模型

| 角色 | 标识 | 能力边界 |
|------|------|----------|
| 管理员 | `ADMIN` | 删除/修改所有人内容；审核文物与来信；下架任意商品 |
| 审核员 | `MODERATOR` | 将他人文章/文物/来信「下线·驳回 / 恢复」 |
| 普通用户 | `USER` | 管理自己的内容；发布闲置；下单购买 |

审核流统一模式（博物馆、来信共用）：**用户提交 → 待审(PENDING) → 审核员通过/驳回 → 通过后方可公开**。
古董铺：下单即锁库存防超卖，且禁止买自己的商品。

> 安全铁律：权限校验写在 **Service 层**，前端隐藏按钮只是体验，真正拦截在后端。

## 4. 模块逐块详解

### 4.1 用户与鉴权
- **功能**：注册、登录、注销（服务端令牌黑名单）、修改昵称/邮箱/头像/密码。
- **入口**：`login.html`、`register.html`、`profile.html`。
- **核心接口**：`POST /api/auth/register`、`POST /api/auth/login`、`POST /api/auth/logout`、`GET|PUT /api/user/info`。
- **技术亮点**：JWT（`jjwt 0.12.5`）+ 拦截器统一鉴权；密码 **MD5 + 随机盐**；头像以 base64 存 `MEDIUMTEXT`。
- **演示要点**：注册新账号 → 登录 → 在个人中心上传头像 → 刷新顶栏头像更新。

### 4.2 文章与热点
- **功能**：发布/存草稿、编辑、删除、分类筛选、关键词搜索、分页、点赞（按用户去重）、热点排行榜。
- **入口**：`index.html`（首页/热点）、`article.html`（详情）、`editor.html`（写文章）。
- **核心接口**：`GET /api/articles`（列表）、`GET /api/articles/{id}`、`POST /api/articles`、`PUT|DELETE /api/articles/{id}`、`PATCH /api/articles/{id}/status`（下线/恢复）、`POST /api/articles/{id}/like`、`GET /api/hot`。
- **技术亮点**：`PageResult` 统一分页；`ArticleStatus` 三态（草稿/已发布/已下线）；点赞去重防刷；MODERATOR 可内容审核。
- **演示要点**：写一篇文章 → 首页看到 → 点赞 → 热点榜上升；用 moderator 账号把文章下线再恢复。

### 4.3 线上博物馆
- **功能**：文物画廊（画框卡片 + 馆藏编号）、提交文物（带图）、审核流、详情展板（左图右文 + 策展人笔记 + 审核备注）、**问策展人** AI。
- **入口**：`museum.html`、`inbox.html`（审核信箱）。
- **核心接口**：`GET /api/relics`、`GET /api/relics/{id}`、`POST /api/relics`、`GET /api/relics/inbox`、`POST /api/relics/{id}/review`、`DELETE /api/relics/{id}`。
- **技术亮点**：复古学术风视觉（花饰标题、编号式陈列）；提交即进 `PENDING`；详情弹窗信息分层（Curator's Note / Review Note）；复用 AI 通道做「问策展人」。
- **演示要点**：普通用户提交一件文物 → 刷新页面不显示（待审）→ 用 admin 在 `/inbox` 通过 → 画廊出现 → 点开详情「问策展人」。

### 4.4 读者来信
- **功能**：来信选登、提交来信（可留联系方式）、审核流、管理员信箱。
- **入口**：`letters.html`、`inbox.html`。
- **核心接口**：与博物馆同构（`/api/letters` 系列：`GET /`、`GET /mine`、`GET /{id}`、`POST /`、`GET /inbox`、`POST /{id}/review`、`DELETE /{id}`）。
- **技术亮点**：完整复用 Relic 审核范式，开发成本极低、行为一致；体现「模式复用」工程思想。
- **演示要点**：提交一封读者来信 → 审核通过 → 选登列表展示。

### 4.5 每日谜题
- **功能**：每日 5×5 对称词方、点击「开始答题」后计时、检查/显示答案、提交成绩、今日榜单。
- **入口**：`puzzle.html`。
- **核心接口**：`GET /api/puzzles/today`、`GET /api/puzzles/leaderboard`、`POST /api/puzzles/scores`、`GET /api/puzzles/scores/mine`。
- **技术亮点**：内置 2 套对称词方题库，按 `年×370+年内天数` 取模选今日词方（同天全局一致、隔天轮换）；每人每天每词方只留最小秒数；**开始答题才计时**（覆盖层 + 输入禁用）。`crossword.js` 已重构为 `buildCrossword()` 工厂，首页与谜题页共用。
- **演示要点**：打开页面计时停在 00:00 → 点「开始答题」开始走 → 填完检查 → 提交成绩 → 榜单出现名次。

### 4.6 版画工坊（纯前端）
- **功能**：上传图片 → 浏览器内 Canvas 生成「老照片」（sepia + 半调 + 颗粒 + 暗角）或「木刻线描」（Sobel 边缘）两种风格 → 下载 PNG。
- **入口**：`engraving.html`。
- **核心接口**：**无后端**，全部在 `engraving.js` 用 Canvas 处理。
- **技术亮点**：零后端依赖、纯前端图像处理；展示前端也能做「重」计算。
- **演示要点**：选一张照片 → 切两种风格 → 下载成品。

### 4.7 古董铺闲置商城
- **功能**：闲鱼式二手集市——发布闲置（带图/价格/分类/成色）、在售浏览、分类筛选、关键词搜索、点商品 → 立即购买 → 订单确认 → 提交订单 → 付款页（**付款按钮按需求置灰**）；「我买的 / 我卖的」订单管理。
- **入口**：`shop.html`、`product.html`、`checkout.html`、`orders.html`、`payment.html`。
- **核心接口**：`GET /api/shop/products`（公开）、`GET /api/shop/products/{id}`（公开）、`POST /api/shop/products`、`GET /api/shop/products/mine`、`DELETE /api/shop/products/{id}`、`POST /api/shop/orders`、`GET /api/shop/orders/mine`、`GET /api/shop/orders/sold`、`GET /api/shop/orders/{id}`。
- **技术亮点**：下单即**快照商品信息并标记售出锁库存**防超卖；禁止买自己的商品；有待付订单时禁止下架；预置 4 件示例闲置一打开即可体验；付款页按需求保持 `disabled`。
- **演示要点**：用 alice 发布一件闲置 → 用另一个账号打开商品 → 立即购买 → 填收货信息 → 提交订单 → 跳付款页（按钮灰色提示「支付通道暂未开通」）→ 在「我的订单」看到「我买的 / 我卖的」。

### 4.8 AI 助手 / 策展人
- **功能**：右侧 AI 助手浮窗（全站可唤出）、博物馆「问策展人」、文章「一句话导读」。
- **入口**：`article.html`（浮窗 + 导读）、`museum.html`（问策展人）。
- **核心接口**：`POST /api/ai/chat`（需登录，按用户名隔离记忆）、`GET /api/ai/query`（🔓 内部检索接口，供 Python 服务用）。
- **技术亮点**：`AiService` 双模式——**优先直连 OpenRouter**（配 key 即真实调用，按 username 维护最近 10 轮历史），失败回退本地 Python ai-assistant 代理；前端构造领域 prompt 复用同一通道，零新增后端。
- **演示要点**：文章页点「一句话导读」生成摘要；博物馆详情点「问策展人」提问；右侧浮窗自由对话。

### 4.9 图片 / 头像系统
- **功能**：用户头像、文物图、商品图统一走「前端压缩 → base64 → 入库」。
- **入口**：profile / museum 提交 / shop 发布 的上传控件。
- **核心接口**：随所属模块（User/Relic/Shop 的写接口）。
- **技术亮点**：`common.js` 的 `fileToDataURL()` 用 Canvas 等比缩放 + JPEG 压缩（头像 240px、文物/商品 900px、质量 0.82）；字段 `MEDIUMTEXT` 防溢出；后端设 base64 长度上限防库撑爆；`isImageSrc()` 渲染时容错（无图/旧路径自动占位）。
- **演示要点**：任意上传处选图 → 实时预览压缩后的效果 → 保存后数据库存的是 base64 文本。

## 5. 前端公共能力（给前端协作同学）

- **`js/api.js`**：`api(path, options)` 封装——自动带 JSON 头、解析 `Result`、处理 `code!==0` 与 `401`（跳登录）。所有模块调用都走它，不要在页面里裸写 `fetch`。
- **`js/common.js`**：`setupHeader()` 渲染顶栏（登录态/头像/导航高亮）、`ICON_DEFS` SVG 图标库、`fileToDataURL()` 图片压缩、`isImageSrc()` 图片判定。
- **视觉令牌**：`css/style.css` 定义报纸学术风设计变量（`--bg/--text/--accent/--rule/--serif` 等），`css/modules.css` 按模块组织样式，支持明/暗主题。
- **版本号**：静态资源统一带 `?v=36`，改前端后改版本号即可破浏览器缓存。

## 7. 本地部署与答辩演示步骤

1. 安装并启动 **MySQL 8.x**，确认 `application.yml` 中 `datasource.username/password` 与本机一致。
2. IDEA 打开 `blog-springboot`，Maven 自动导入；运行 `BlogApplication`。
3. 浏览器开 `http://localhost:8080/`。
4. 用体验账号演示（admin/admin123、moderator/mod123、alice/user123）。
5. 推荐演示动线：
   - 首页文章 + 热点榜 → 写文 + 点赞；
   - 博物馆提交文物 → 切 admin 在 `/inbox` 审核 → 画廊出现 → 问策展人；
   - 读者来信提交 → 审核 → 选登；
   - 每日谜题：开始答题 → 计时 → 提交 → 榜单；
   - 版画工坊：上传图 → 两种风格 → 下载；
   - 古董铺：发布闲置 → 另一账号购买 → 订单 → 付款页（置灰）；
   - AI：文章一句话导读 / 右侧浮窗对话。
6. （可选）AI 真实调用：在 `application.yml` 填 `openrouter.api-key` 后重启；否则需启动本地 Python ai-assistant（8000 端口）。

## 9. 已知限制与后续迭代

- **密码**：MD5 仅学习用，生产换 BCrypt。
- **图片存库**：base64 膨胀 ~33%，上量建议改对象存储 + 库存路径。
- **付款**：`payment.html` 按钮按需求置灰，未接支付网关（`ShopService` 已预留改造点）。
- **AI**：`openrouter/free` 免费路由模型不固定、有限流；可换具体 `:free` 模型或接本地 Python 服务。
- **注销黑名单**：内存实现，集群需换 Redis。
- **扩展方向**：评论区、站内私信、商品评论、支付对接、图片 CDN、WebSocket 实时通知。
