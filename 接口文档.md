# M.E.S.S Journal —— 接口文档（API Reference）

> 配套文档：[功能说明文档.md](./功能说明文档.md)（讲功能与模块划分）
> 本文档精确到每个接口的 **方法 / 路径 / 权限 / 参数 / 请求体 / 响应示例**。

---

## 通用说明

### Base URL
```
http://localhost:8080/api
```

### 鉴权方式
- 登录后，前端自动在请求头附加：`Authorization: Bearer <token>`
- **公开接口（无需 token）**：
  - `POST /api/auth/register`、`POST /api/auth/login`、`POST /api/auth/logout`
  - `GET /api/articles*`（列表/详情，只读；带 token 则注入用户信息）
  - `GET /api/hot*`（热点榜，只读）
- **其余 `/api/**` 接口必须携带合法 token**，否则返回：
  ```json
  { "code": 401, "message": "未登录或 token 缺失" }
  ```
- token 注销（黑名单）后访问受保护接口：返回 `401 登录已失效，请重新登录`。

### 统一响应结构
```json
{ "code": 0, "message": "ok", "data": {} }
```
- 成功：`code == 0`，`data` 为业务数据。
- 失败：`code != 0`，`message` 为错误描述（如 `401 未登录`、`403 无权修改该文章`、业务提示 `用户名已存在`）。
- 分页接口 `data` 为 `PageResult`：`{ list, total, page, size, totalPages }`。

### 角色（Role）
| 角色 | 说明 |
| --- | --- |
| `ADMIN` | 最高权限，可改/删/下线/恢复所有文章 |
| `MODERATOR` | 审核员，可下线/恢复他人文章，不能删除 |
| `USER` | 普通用户，仅管理自己的文章、可点赞 |

## 1. 认证模块 `/api/auth`

### 1.1 用户注册
- **接口**：`POST /api/auth/register`
- **权限**：公开
- **请求体**：
```json
{
  "username": "bob",          // 必填，唯一
  "password": "user123",      // 必填
  "nickname": "鲍勃",          // 选填，缺省用 username
  "email": "bob@blog.com"     // 选填
}
```
- **成功响应**：
```json
{ "code": 0, "message": "注册成功", "data": null }
```
- **失败示例**：`{ "code": 500, "message": "用户名已存在", "data": null }`

### 1.2 用户登录
- **接口**：`POST /api/auth/login`
- **权限**：公开
- **请求体**：
```json
{ "username": "alice", "password": "user123" }
```
- **成功响应**（`data` 含 token 与脱敏用户视图）：
```json
{
  "code": 0, "message": "ok",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "user": {
      "id": 3, "username": "alice", "nickname": "爱丽丝",
      "email": "alice@blog.com", "avatar": "", "role": "USER",
      "createdAt": "2026-07-17T10:00:00"
    }
  }
}
```
- **失败示例**：`{ "code": 500, "message": "密码错误", "data": null }`

### 1.3 用户注销
- **接口**：`POST /api/auth/logout`
- **权限**：需登录（token 有效）
- **说明**：把当前 token 的 `jti` 加入黑名单，立即失效。
- **成功响应**：
```json
{ "code": 0, "message": "已注销", "data": null }
```

## 2. 用户模块 `/api/user`

### 2.1 获取当前用户信息
- **接口**：`GET /api/user/info`
- **权限**：需登录
- **成功响应**（脱敏视图，不含密码/盐）：
```json
{
  "code": 0, "message": "ok",
  "data": {
    "id": 3, "username": "alice", "nickname": "爱丽丝",
    "email": "alice@blog.com", "avatar": "", "role": "USER",
    "createdAt": "2026-07-17T10:00:00"
  }
}
```

### 2.2 修改当前用户信息
- **接口**：`PUT /api/user/info`
- **权限**：需登录
- **请求体**（字段均可选；改密码时 `oldPassword` + `newPassword` 必填且需原密码正确）：
```json
{
  "nickname": "新昵称",
  "email": "new@blog.com",
  "avatar": "https://...",
  "oldPassword": "user123",
  "newPassword": "newPass456"
}
```
- **成功响应**：返回更新后的脱敏用户视图（同 2.1 结构）。
- **失败示例**：`{ "code": 500, "message": "原密码错误", "data": null }`

## 3. 文章模块 `/api/articles`

### 3.1 文章列表（分页 + 筛选）
- **接口**：`GET /api/articles`
- **权限**：半公开（无 token 可匿名浏览；带 token 注入用户信息）
- **查询参数**：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `category` | string | 否 | 分类名精确匹配 |
| `keyword` | string | 否 | 标题/摘要/正文模糊搜索 |
| `page` | int | 否 | 页码，默认 `0`（从 0 起） |
| `size` | int | 否 | 每页条数，默认 `8` |
| `mine` | boolean | 否 | `true` 只看当前用户（含草稿），需登录；默认 `false` |

- **成功响应**（公开时 `data.list` 仅含 `PUBLISHED`）：
```json
{
  "code": 0, "message": "ok",
  "data": {
    "list": [
      {
        "id": 2, "title": "SpringBoot 拦截器最佳实践", "summary": "...",
        "category": "技术", "status": "PUBLISHED", "authorId": 4,
        "authorName": "鲍勃", "likeCount": 128,
        "createdAt": "2026-07-10T...", "publishedAt": "2026-07-10T..."
      }
    ],
    "total": 4, "page": 0, "size": 8, "totalPages": 1
  }
}
```
- **注意**：`mine=true` 但未登录 → `{ "code": 401, "message": "请先登录", "data": null }`

### 3.2 文章详情
- **接口**：`GET /api/articles/{id}`
- **权限**：半公开
- **路径参数**：`id`（文章 ID）
- **可见性规则**：
  - `PUBLISHED`：所有人可见。
  - `DRAFT`：仅作者本人可见，否则 `403 无权访问该草稿`。
  - `OFFLINE`：仅作者或 MODERATOR/ADMIN 可见，否则 `403 该文章已被下线，暂不可见`。
- **成功响应**（`data` 含文章与当前用户点赞状态）：
```json
{
  "code": 0, "message": "ok",
  "data": {
    "article": {
      "id": 2, "title": "SpringBoot 拦截器最佳实践", "content": "...",
      "summary": "...", "category": "技术", "status": "PUBLISHED",
      "authorId": 4, "authorName": "鲍勃", "likeCount": 128,
      "createdAt": "...", "publishedAt": "..."
    },
    "liked": false
  }
}
```

### 3.3 发布 / 存草稿
- **接口**：`POST /api/articles`
- **权限**：需登录
- **请求体**：
```json
{
  "title": "我的第一篇",        // 必填
  "content": "正文内容",        // 选填
  "summary": "摘要",           // 选填
  "category": "随笔",          // 选填
  "draft": false               // true=存草稿；false/缺省=直接发布
}
```
- **成功响应**：返回新建的 `Article` 对象（含 `id`、`status` 等）。
- **失败示例**：`{ "code": 500, "message": "标题不能为空", "data": null }`

### 3.4 编辑文章
- **接口**：`PUT /api/articles/{id}`
- **权限**：作者本人 或 ADMIN
- **路径参数**：`id`
- **请求体**：同 3.3（字段均可选；`draft` 由 true 变 false 时自动补全发布时间）。
- **成功响应**：返回更新后的 `Article`。
- **失败示例**：`{ "code": 403, "message": "无权修改该文章", "data": null }`

### 3.5 删除文章
- **接口**：`DELETE /api/articles/{id}`
- **权限**：作者本人 或 ADMIN（MODERATOR 不能删）
- **成功响应**：`{ "code": 0, "message": "删除成功", "data": null }`

### 3.6 下线 / 恢复
- **接口**：`PATCH /api/articles/{id}/status`
- **权限**：仅 MODERATOR 与 ADMIN
- **路径参数**：`id`
- **请求体**：
```json
{ "status": "OFFLINE" }    // OFFLINE=下线；PUBLISHED=恢复
```
- **成功响应**：返回状态更新后的 `Article`。
- **失败示例**：`{ "code": 403, "message": "无权限进行下线/恢复操作", "data": null }`

### 3.7 点赞 / 取消点赞
- **接口**：`POST /api/articles/{id}/like`
- **权限**：需登录
- **路径参数**：`id`
- **说明**：Toggle 模式——未点赞则点赞，已点赞则取消；按用户去重；仅 `PUBLISHED` 文章可点赞。
- **成功响应**：
```json
{ "code": 0, "message": "ok", "data": { "likeCount": 129, "liked": true } }
```
- **失败示例**：`{ "code": 500, "message": "只有已发布文章可点赞", "data": null }`

## 4. 热点模块 `/api/hot`

### 4.1 热点榜
- **接口**：`GET /api/hot`
- **权限**：半公开（只读）
- **查询参数**：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `limit` | int | 否 | 返回条数，默认 `10` |

- **排序**：`PUBLISHED` 文章 → 按 `likeCount` 倒序（相同则按 `publishedAt` 倒序）。
- **成功响应**（`data` 为 `Article` 数组）：
```json
{
  "code": 0, "message": "ok",
  "data": [
    { "id": 2, "title": "SpringBoot 拦截器最佳实践", "likeCount": 128, "category": "技术", "status": "PUBLISHED" },
    { "id": 4, "title": "热点榜是怎么算出来的", "likeCount": 73, "category": "技术", "status": "PUBLISHED" }
  ]
}
```

## 5. AI 助手模块 `/api/ai`

### 5.1 AI 问答
- **接口**：`POST /api/ai/chat`
- **权限**：需登录
- **请求体**：
```json
{ "question": "请用一句话解释什么是 JWT" }
```
- **成功响应**（`data` 为模型回答文本）：
```json
{ "code": 0, "message": "ok", "data": "JWT 是一种带签名、自包含的令牌，服务端用它无状态地校验用户身份。" }
```
- **未配置 Key**：返回友好提示文本（非报错），前端照常展示。
- **失败示例**：`{ "code": 500, "message": "调用 AI 失败：...", "data": null }`
- **架构**：Spring Boot 仅做代理转发（`AiController` 把登录用户名作为 `session_id` 转发到 `ai.python-url` 指向的 Python 服务）；真正的模型调用与工具调度由 Python + LangChain 完成。
- **能力**：按用户隔离的近 10 轮记忆；可自动调用站内检索工具（`search_website`，回答“在哪修改密码”等）与天气查询工具（`get_weather`）。
- **配置说明**：外部大模型密钥/模型在 `application.yml` 的 `openrouter` 段（由 Python 服务读取）；Python 服务地址在 `ai.python-url`（默认 `http://localhost:8000/api/ai/chat`）。

## 附：常见错误码 / 业务消息速查

| code | message 示例 | 含义 / 触发条件 |
| --- | --- | --- |
| 0 | `ok` | 成功 |
| 401 | `未登录或 token 缺失` | 受保护接口未带 token |
| 401 | `登录已失效，请重新登录` | token 已在注销黑名单中 |
| 401 | `请先登录` | 文章列表 `mine=true` 但未登录 |
| 403 | `无权修改该文章` / `无权删除该文章` | 非作者且非 ADMIN |
| 403 | `无权限进行下线/恢复操作` | 非 MODERATOR/ADMIN |
| 403 | `该文章已被下线，暂不可见` | 访问已下线文章且无权限 |
| 403 | `无权访问该草稿` | 访问他人草稿 |
| 500 | `用户名已存在` / `密码错误` / `原密码错误` / `标题不能为空` | 业务校验失败 |

> 说明：本文档中所有接口路径、参数、权限均依据当前工程代码（控制器 / 拦截器 / 服务层）整理，与实现保持一致。
